## 2. Terminology

"v1.1" is the **internal** name for the hardened benchmark (sanitized public
model-facing views, pattern-bound gold labels, balanced hard-split decision
schedule). The paper-facing and public-release name is simply **ReTrace-Bench**.
v1.0 is the earlier/pilot state and becomes a legacy/pilot appendix only — this
report and the cleanup do **not** narrate a public v1.0 → v1.1 history.

**Important:** the committed `main/hard/realistic/calibration/private_hidden`
splits under `data/retrace_bench/` were generated by an *older* generator and
are **stale** relative to the current validator/scorer: re-validating them
produces thousands of errors (pattern-name leakage in `expected_answer`,
pattern/failure-mode binding mismatches, realistic auto-marked `reviewed`). The
**current** generator (`scripts/generate_retrace_bench_final.py` +
`benchmark/retrace_bench/generation/hard_plus_blueprints.py`) produces
validator-clean output (verified on a `--smoke` run), so the fix is to
**regenerate** deterministically, not to hand-patch the stale rows.

(gated DeepSeek/GLM/Kimi on `hard_300` and `main_3000_first500`).

## 6. Known failing tests (baseline, before this pass)

`pytest tests/retrace_bench -q` → **7 failed, 76 passed**. These are
**pre-existing on `benchmark`**, not introduced here:

A-class (engine/packaging):
- `test_general_benchmark::test_general_blueprint_render_validate` — the legacy
  general blueprint render (`generate_retrace_bench_blueprints.blueprint` +
  `render_retrace_bench_dataset.render_one`) produces rows with **no pattern
  metadata**, so `validate_pattern_semantics` rejects all 40. (A2)
- `test_hf_package_readme::test_package_script_template_has_calibration_and_realistic_semantics`
  — HF package template auto-marks realistic `annotation_status = reviewed`
  instead of `pending`. (A1)
- `test_prediction_evaluator_cli::test_cli_scores_sample_predictions` — sample
  predictions fixture does not match the sample scenarios (160 "extra prediction
  with no matching scenario"). (A3)

B-class (mixed v1.0/v1.1 canonicalization):
- `test_retrace_data_splits::test_only_canonical_splits_present` — extra dirs
  (`hard_300_en`, `realistic_100_en`, …) present. (B4)
- `test_retrace_data_splits::test_public_split_label_matches` — v1.1 rows use
  `public_split_name`, not `split`. (B2)
- `test_retrace_data_splits::test_memory_and_event_ids_disjoint_across_splits`
  — shared event ids across stale splits. (B3)
- `test_retrace_data_splits::test_all_rows_parse_and_have_required_fields` —
  `main_3000_en` rows miss the legacy `tasks`/`split`/`secondary_failure_modes`
  fields the v1.0 test expects. (B1/B2)

## 7. What will be fixed in this pass

1. A-class: pattern-bound general blueprint render; HF package template realistic
   `annotation_status = pending` (drop fabricated "reviewed"/human claim); CLI
   sample-prediction/scorer fixture alignment.
2. B-class: make `data/retrace_bench_v1_1/` the canonical layout; move v1.0 to a
   clearly-named legacy location; repoint tests at v1.1; stop testing v1.0 as
   canonical.
3. Regenerate all five v1.1 splits deterministically (seed `2027`).
4. Validate every split; gold-oracle every split (must be 1.0 on core metrics).
5. Run **offline** baselines only.
6. Human validation/annotation protocol + materials + scoring scripts.
7. Statistical reporting note.
8. Hugging Face v1.1 release bundle (**not** published).
9. Repository cleanup plan.

## 8. What will NOT be done in this pass

- **No model/API reruns** (budget + quota). No paid or external model calls.
- **No training** of any model. ReTrace-Learn untouched.
- **No fabricated** model results or human-validation results.
- **No Hugging Face publish** (bundle prepared only).
- Lost hard500 model numbers are **not** promoted to citable results.
- No secrets committed (`.env`, API keys, raw API logs excluded).
